# Алгоритм индексатора

## Вход

- Переменные окружения процесса
- Значение `PATH`
- `aliases.json`

## Шаги

1. Создать/проверить директорию данных и файлы (`index.sqlite`, `aliases.json`, `cache.json`).
2. Очистить таблицу `entries`.
3. Для каждой ENV-переменной:
   - если значение — абсолютный существующий путь, добавить запись (`source=env`).
4. Для каждого элемента `PATH`:
   - если это абсолютная существующая директория, добавить запись (`source=path`, `name=PATH_N`).
5. Для каждого пользовательского алиаса:
   - если путь абсолютный и существует, добавить запись (`source=alias`).
6. Для каждой записи сформировать `keywords` из:
   - имени (в нижнем регистре)
   - частей пути.
7. Записать записи в SQLite c `INSERT OR IGNORE`.

## Поиск

Запрос ищет совпадения по `name`, `path`, `keywords` и ранжирует:

1. точное совпадение имени;
2. префикс имени;
3. более короткий путь.

## Псевдопуть

Для входного абсолютного пути:

1. найти все алиасы/ENV, чьи пути являются префиксом;
2. выбрать самый длинный префикс;
3. заменить префикс на `$ALIAS`.
